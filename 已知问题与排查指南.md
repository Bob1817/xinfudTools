# 已知问题与排查指南 (Known Issues & Troubleshooting)

本文档记录了项目开发和打包过程中遇到的典型错误及其解决方案，便于日后快速排查。

---

## 🐛 1. Windows 打包版生成报表时闪退 (FlushError)

**报错信息**：
```text
Traceback (most recent call last):
  File "C:\Program Files (x86)\HRTool\_internal\ui\tools\tax_merge_page.py", line 438, in _start_generate
    sys.stdout.flush()
    ^^^^^^^^^^^^^^^^
AttributeError: 'NoneType' object has no attribute 'flush'
```

**发生场景**：
在 Windows 安装版（使用 PyInstaller `--windowed` 或 `--noconsole` 打包）中，点击“生成个税申报表”时程序瞬间退出，无任何报错弹窗。

**根本原因**：
在无控制台模式（Windowed Mode）下，Python 标准输出 `sys.stdout` 和标准错误 `sys.stderr` 会被设置为 `None`。
代码中为了调试遗留的 `print(..., flush=True)` 或 `sys.stdout.flush()` 语句试图操作一个空对象，引发 `AttributeError` 导致进程终止。

**修复方案**：
1.  **全局重定向 (main.py)**：在程序启动时强制将 `stdout/stderr` 重定向到 `io.StringIO()`。
    ```python
    import sys, io
    if sys.stdout is None: sys.stdout = io.StringIO()
    if sys.stderr is None: sys.stderr = io.StringIO()
    ```
2.  **清理代码**：移除所有生产环境中的 `print()` 和 `flush()` 调用，改用安全的日志记录（如写入文件）。

**涉及版本**：
- 影响版本：v1.0.9, v1.0.10
- 修复版本：v1.0.11

## 🐛 2. 程序启动即闪退 (TypeError)

**报错信息**：
```text
TypeError: setHighDpiScaleFactorRoundingPolicy(policy: Qt.HighDpiScaleFactorRoundingPolicy): argument 1 has unexpected type 'ColorScheme'
```

**发生场景**：
双击安装包安装后的程序图标，程序无任何反应或闪退，无法进入主界面。

**根本原因**：
在 `main.py` 中尝试手动配置高 DPI 缩放策略时，错误地将 `QApplication.styleHints().colorScheme()`（返回 `ColorScheme` 枚举）作为参数传给了 `setHighDpiScaleFactorRoundingPolicy`（需要 `HighDpiScaleFactorRoundingPolicy` 枚举），导致类型不匹配。

**修复方案**：
移除错误的配置代码，使用 Qt 默认的 DPI 行为。
```python
# 错误代码（已删除）：
# QApplication.setHighDpiScaleFactorRoundingPolicy(QApplication.styleHints().colorScheme())

# 修正：直接移除该行，保持默认即可
```

**涉及版本**：
- 影响版本：v1.0.9
- 修复版本：v1.0.10

## 🐛 3. 检查更新失败 (API 404/403)

**报错信息**：
- Gitee 检查更新失败：HTTP Error 404: Not Found
- GitHub 检查更新失败：HTTP Error 403: Forbidden

**发生场景**：
在设置页面点击“检查更新”时，提示无法连接更新服务器。

**根本原因**：
1.  **API 路径错误**：早期代码使用了 `/releases/latest` 端点，但该端点在某些网络环境或仓库状态下不稳定。
2.  **仓库状态**：如果仓库被设为私有 (Private) 或 Release 处于草稿 (Draft) 状态，未认证的 API 请求会返回 404 或 403。

**修复方案**：
1.  **更改 API 请求**：改用 `?per_page=1` 获取 Release 列表的第一个元素，提高成功率。
2.  **友好提示**：当检测到错误时，弹窗提示用户前往 Release 页面手动下载，而不是直接崩溃或仅显示技术性错误代码。

**涉及版本**：
- 影响版本：v1.0.1 ~ v1.0.4
- 修复版本：v1.0.6 (优化 API 逻辑), v1.0.9 (优化错误提示)

## 💡 通用排查建议

如果未来遇到 **Windows 闪退** 问题，请按以下步骤操作：

1.  **查看日志文件**：程序通常会在 `C:\Users\用户名\Desktop\HR_Tools_Crash_Debug.txt` 或 `C:\Users\用户名\Documents\HR_Tools_Crash.log` 生成崩溃日志。
2.  **命令行运行**：在 CMD 中通过 `start_windows.bat` 或直接运行 `python main.py` 启动，可以直接在终端看到 Python 报错堆栈信息。
