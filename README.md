## 项目概述

一个运行在 Windows 上的本地客户端工具集，用于解决人力资源日常办公中的表格处理问题。

**当前版本**: [v1.0.11](../VERSION.md) | [更新日志](../CHANGELOG.md) | [路线图](../ROADMAP.md)

### 核心功能

- **工具 1**：发薪表 → 个税扣缴申报表（单表模式）
- **工具 2**：社保数据表 + 发薪表 → 个税扣缴申报表（双表合并模式）
- **历史记录**：自动保存生成记录，支持查看和管理
- **自动更新**：支持 Gitee/GitHub 双源自动检查更新

## 安装与运行

### 环境要求

- Windows 10/11
- Python 3.10+
- pip

### 安装依赖

```bash
pip install -r requirements.txt
```

### 运行应用

#### Mac 系统

```bash
# 方式 1: 使用运行脚本
./run_mac.sh

# 方式 2: 手动运行
python3 main.py
```

#### Windows 系统

```bash
python main.py
```

### 测试应用

#### Mac 系统

```bash
# 运行完整测试
./test_mac.sh
```

使用提供的脚本打包为 `.app` 文件：

```bash

```

打包完成后，在 `dist` 目录中找到 `HR 数据处理工具.app`


打包完成后，在 `dist` 目录中找到 `HR 数据处理工具.exe`

## 核心业务：工具 2 — 社保表 + 发薪表 → 个税申报表

### 输入 1：社保数据表

社保数据表包含多个 sheet，其中**【本月发薪明细】**sheet 包含社保明细数据：

~~~text
列名                  说明
──────────────────────────────────────
序号
姓名              ← 核心字段
身份证号          ← 核心字段，用于关联发薪表
银行卡号
银行名称
手机号码
收入
*个人养老*        ← 提取字段
*个人医疗*        ← 提取字段
*个人失业*        ← 提取字段
*个人公积金*      ← 提取字段
累计专项附加扣除
扣除合计
应发金额
个税
税后补扣
实发金额
~~~

### 输入 2：发薪表

发薪表结构与工具 1 相同（38 列），包含应发金额、个税、企业/个人社保等数据。

### 合并逻辑

1. 以**身份证号**为关键字段，匹配社保表和发薪表
2. 优先使用社保表中的个人社保数据（养老、医疗、失业、公积金）
3. 若社保表中无匹配记录，则使用发薪表个人部分数据
4. 生成标准的 28 列个税申报表

### 输出：个税扣缴申报表 Excel

申报表结构与工具 1 相同（28 列），数据来源：

| 申报表字段 | 数据来源 |
|-----------|---------|
| 工号 | 自动生成序号 |
| 姓名 | 发薪表.姓名 |
| 证件号码 | 发薪表.身份证号（清洗后） |
| 本期收入 | 发薪表。应发金额 |
| 基本养老保险费 | 优先：社保表。个人养老；备选：发薪表个人部分养老 |
| 基本医疗保险费 | 优先：社保表。个人医疗；备选：发薪表个人部分医疗 |
| 失业保险费 | 优先：社保表。个人失业；备选：发薪表个人部分失业 |
| 住房公积金 | 优先：社保表。个人公积金；备选：发薪表个人部分公积金 |
| 通讯费用 | 发薪表。通讯费 |
| 备注 | 发薪表。备注 |

## 使用说明

### 工具 1：发薪表 → 个税申报表（单表模式）

适用场景：发薪表已包含完整的个人社保公积金数据

1. 打开应用，切换到"工具 1"Tab
2. 选择发薪数据表 Excel 文件
3. 输入所属期（如 2025-04）
4. 点击"生成个税申报表"
5. 申报表将保存到桌面

### 工具 2：社保表 + 发薪表 → 个税申报表（双表合并模式）

适用场景：需要以社保表中的个人社保数据为准生成申报表

1. 打开应用，切换到"工具 2"Tab
2. 选择社保数据表（包含【本月发薪明细】sheet）
3. 选择发薪数据表
4. 输入所属期（如 2025-04）
5. 系统自动显示匹配情况：
   - 身份证号匹配数量
   - 仅在发薪表/社保表的数据
6. 点击"生成个税申报表"
7. 申报表将保存到桌面

### 历史记录

1. 切换到"历史记录"Tab
2. 查看生成记录
3. 可打开文件、打开文件夹或删除记录

## 核心业务：工具 1 — 发薪表 → 个税申报表

### 输入：发薪表 Excel

发薪表共 **38 列**，结构如下（按列顺序）：

~~~text
位置 		列名 				说明
────────────────────────────────────────────── 
0 		发薪月份 
1 		完成时间 
2 		姓名 					← 核心字段 
3 		身份证号 			 ← 核心字段，需清洗（去空格、去.0、转大写） 
4 		手机号码 
5 		项目名称 
6 		客户公司 
7 		银行名称 
8 		银行卡号 
9 		应发金额 			 ← 核心字段 
10 		个税 					← 核心字段 
────────────────────────────────────────────── 
11~17 企业部分 社保公积金总计 | 养老 | 失业 | 工伤 | 医疗 | 公积金 | 总计 
────────────────────────────────────────────── 
18~23 个人部分 养老 | 失业 | 工伤 | 医疗 | 公积金 | 总计 
────────────────────────────────────────────── 
24 		实发金额 
25 		通讯费 
26 		税前抵扣项 
27 		税后补扣补退 
28 		个税差额调整 
29 		备注 
30 		适用账套 
31 		服务产品 
32 		发放状态 
33 		失败原因 
34 		结算单名称 
35 		结算人员 
36 		服务主体 
37 		(可能有更多列)
~~~

> **关键区分**：列 11~17 为企业部分，列 18~23 为个人部分。两组列名中均包含"公积金"（原为"生育"，已替换）。

### 输出：个税扣缴申报表 Excel

申报表共 **28 列**，结构如下：

~~~text
*证件类型 							 固定值: "居民身份证" 
*证件号码 							 发薪表.身份证号 
本期收入 								 发薪表.应发金额 
本期免税收入 							0 
基本养老保险费 					 个人部分[0] = 养老 
基本医疗保险费 					 个人部分[3] = 医疗 
失业保险费 		 					 个人部分[1] = 失业

累计子女教育 							0 
累计继续教育 							0 
累计住房贷款利息 	 				 0 
累计住房租金 							0 
累计赡养老人 							0 
累计3岁以下婴幼儿照护 		  0 
累计个人养老金 						0 
企业(职业)年金 						 0 
商业健康保险 							 0 
税延养老保险 							 0 
公务交通费用 							 0 
通讯费用 									发薪表.通讯费 
律师办案费用 							 0 
西藏附加减除费用 						0 
其他 											0 
准予扣除的捐赠额 						0 
减免税额 									 0 
备注 											发薪表.备注
~~~

### 个人部分列索引映射

~~~text
发薪表个人部分列顺序（索引18~23）: 
[0] 养老 → 申报表.基本养老保险费 
[1] 失业 → 申报表.失业保险费 
[2] 工伤 → 不使用（申报表无此字段） 
[3] 医疗 → 申报表.基本医疗保险费 
[4] 公积金 → 申报表.住房公积金 
[5] 总计 → 不使用
~~~

### 数据处理流程

~~~text
发薪表 Excel 
		│ 
		▼ 
pandas.read_excel(dtype={"身份证号": str}) 
		│ 
		▼ 
身份证号清洗: strip → upper → 去掉 .0 
		│ 
		▼ 
校验必要列: 姓名、身份证号、应发金额 
		│ 
		▼ 
按列索引提取企业部分(11~17)和个人部分(18~23) 
		│ 
		▼ 
映射到申报表28列格式
工号自增
证件类型固定
个人社保列按顺序取值
无数据字段填 0 
		│
    ▼ 
openpyxl 生成带格式 Excel
蓝色表头、白色字体
交替行背景色
金额列 #,##0.00 格式
冻结首行
自动列宽 
		│ 
		▼ 
保存到桌面 + 记录到 SQLite
~~~

## UI 设计

### 主窗口

- 标题：`HR 数据处理工具集 v1.0.11`
- 最小尺寸：960×700
- 使用 QTabWidget，每个工具一个 Tab

### Tab 1：发薪表 → 个税申报表

| 区域       | 控件                             | 功能                                     |
| ---------- | -------------------------------- | ---------------------------------------- |
| 文件选择区 | QGroupBox                        | 选择文件按钮 + 文件名显示 + 所属期输入框 |
| 数据预览区 | QTableWidget (最大高度200)       | 读取文件后显示前5行 + 列位置信息         |
| 生成按钮   | QPushButton (蓝色、加粗、45px高) | 触发后台线程生成                         |
| 进度区     | QProgressBar(无限) + QLabel      | 显示处理进度                             |
| 结果区     | QTextEdit (只读、最大120px高)    | 显示生成结果或错误信息                   |

### Tab 2：历史记录

| 区域   | 控件                       | 功能                                                   |
| ------ | -------------------------- | ------------------------------------------------------ |
| 标题栏 | QLabel + QPushButton(刷新) |                                                        |
| 表格   | QTableWidget (6列)         | 所属期、数据条数、生成时间、源文件名、输出文件名、操作 |
| 操作列 | 打开 / 文件夹 / 删除       | 打开文件、打开所在文件夹、删除记录                     |

## 数据库设计

```sql
CREATE TABLE merge_records (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    period        TEXT NOT NULL,          -- 所属期，如 "2025-04"
    source_file   TEXT,                   -- 源文件完整路径
    output_file   TEXT,                   -- 输出文件完整路径
    total_records INTEGER,                -- 数据条数
    created_at    TEXT DEFAULT (datetime('now', 'localtime')),
    note          TEXT                    -- 备注
);
```
