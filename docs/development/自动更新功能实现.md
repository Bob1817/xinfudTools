# 自动更新功能实现文档

## 📋 功能概述

为 HR 数据处理工具集实现了完整的自动更新功能，支持：
- ✅ 启动时自动检查更新（静默）
- ✅ 手动检查更新
- ✅ 双更新源支持（Gitee 优先，GitHub 备用）
- ✅ 版本比较和更新提示
- ✅ 下载进度显示
- ✅ 设置页面集成

## 📁 新增文件

### 1. 核心模块

#### `core/version.py`
**功能**: 版本信息定义和管理

**主要函数**:
- `get_version()`: 获取当前版本号
- `get_version_tuple()`: 获取版本号元组（用于比较）
- `compare_versions(v1, v2)`: 比较两个版本号

**使用示例**:
```python
from core.version import get_version, compare_versions

current = get_version()  # "1.0.0"
result = compare_versions("1.1.0", "1.0.0")  # 1 (表示 1.1.0 > 1.0.0)
```

#### `core/updater.py`
**功能**: 自动更新核心逻辑

**主要类**:
- `UpdateChecker`: 检查更新线程
  - 从 Gitee/GitHub Release API 获取最新版本
  - 比较版本号
  - 发送更新可用/无更新/错误信号

- `UpdateDownloader`: 下载更新线程
  - 支持断点续传
  - 实时进度回调
  - 自动确定文件名

- `UpdateManager`: 更新管理器工具类
  - 获取应用路径
  - 重启应用
  - 打开下载页面

**更新源 API**:
- Gitee: `https://gitee.com/shibo8817/xinfud-tools/releases/latest`
- GitHub: `https://api.github.com/repos/Bob1817/xinfudTools/releases/latest`

## 📊 工作流程

### 启动时自动检查更新

```
应用启动
   ↓
初始化 MainWindow
   ↓
调用 _silent_check_update()
   ↓
创建 UpdateChecker 线程
   ↓
连接 Gitee API (优先)
   ↓ 失败
连接 GitHub API (备用)
   ↓
获取最新版本信息
   ↓
比较版本号
   ↓
有新版本？
   ├─ 是 → 弹出 InfoBar 提示 → 用户点击 → 跳转到设置页面
   └─ 否 → 静默继续
```

### 用户手动检查更新

```
用户点击"设置"导航项
   ↓
进入设置页面
   ↓
点击"检查更新"按钮
   ↓
显示"正在检查" InfoBar
   ↓
创建 UpdateChecker 线程
   ↓
获取最新版本信息
   ↓
比较版本号
   ↓
结果处理:
   ├─ 有新版本 → 弹出 UpdateDialog
   │              ↓
   │           用户点击"前往下载"
   │              ↓
   │           打开浏览器下载页面
   │
   ├─ 无更新 → 显示"已是最新版本" InfoBar
   │
   └─ 错误 → 显示错误信息 InfoBar
```

## 🎯 使用方法

### 发布新版本

**步骤**:

1. **更新版本号**
   ```python
   # 编辑 core/version.py
   __version__ = "1.1.0"  # 修改版本号
   ```

2. **提交代码到仓库**
   ```bash
   git add .
   git commit -m "发布 v1.1.0"
   git push
   ```

3. **创建 Gitee Release**
   - 访问: https://gitee.com/shibo8817/xinfud-tools/releases
   - 点击"新建 Release"
   - 填写信息:
     - Tag: `v1.1.0`
     - 标题: `v1.1.0 - 更新说明`
     - 内容: 更新日志和功能说明
     - 附件: 上传打包好的安装包
   - 点击"发布"

4. **用户收到更新提示**
   - 启动应用时自动提示
   - 或手动在设置页面检查

### 用户更新软件

**方式一：自动提示**
1. 启动应用，收到更新通知
2. 点击通知进入设置页面
3. 点击"前往下载"
4. 下载最新版本安装包
5. 关闭应用，运行安装包

**方式二：手动检查**
1. 打开应用
2. 进入"设置"页面
3. 点击"检查更新"
4. 按照提示下载更新

## 🔐 安全说明

### 更新源验证
- 仅从官方仓库（Gitee/GitHub）获取更新
- 使用 HTTPS 加密连接
- 不支持第三方更新源

### 数据安全
- 更新检查只读取版本信息
- 不上传任何用户数据
- 历史记录数据库不受更新影响

### 更新方式
- 当前版本采用"下载-手动安装"方式
- 不会自动替换程序文件（避免文件锁定问题）
- 用户完全控制更新时机

## 🐛 故障排除

### 问题 1: 检查更新失败

**症状**: 显示"无法连接到更新服务器"

**原因**:
- 网络连接问题
- Gitee/GitHub 服务不可用
- 防火墙阻止

**解决方案**:
1. 检查网络连接
2. 尝试访问 Gitee 页面确认服务状态
3. 稍后重试

### 问题 2: 更新提示但不想更新

**解决方案**:
- 直接忽略通知（10秒后自动消失）
- 或点击关闭按钮
- 不会影响当前使用

### 问题 3: 更新后遇到问题

**解决方案**:
1. 查看更新日志了解已知问题
2. 提交 Issue 到 Gitee/GitHub
3. 必要时回退到旧版本

## 📝 技术细节

### 版本比较算法
```python
def compare_versions(v1, v2):
    # 将版本号字符串转为元组: "1.2.3" → (1, 2, 3)
    v1_tuple = tuple(map(int, v1.split(".")))
    v2_tuple = tuple(map(int, v2.split(".")))
    
    # 元组比较: (1, 2, 3) > (1, 1, 0) → True
    if v1_tuple > v2_tuple:
        return 1
    elif v1_tuple < v2_tuple:
        return -1
    else:
        return 0
```

### 异步更新检查
