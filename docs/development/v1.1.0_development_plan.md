## 一、需求分析

### 1.1 核心优化需求

#### 需求1：优化数据来源匹配机制
**优先级**: P0 (关键)

**详细需求**:
1. **身份证号合并逻辑**
   - 以身份证号为唯一识别ID
   - 同一身份证号多条数据时，数值字段累加
   - 姓名、手机号不同时，以第一条数据为准
   - 最终输出以发薪数据表为准（发薪表没有的人员不输出）

2. **社保数据表多Sheet识别**
   - 遍历所有Sheet，自动识别包含社保数据的Sheet
   - 不固定Sheet名称和位置
   - 识别字段：【养老】【医疗】【失业】【公积金】
   - 智能匹配规则：
     - 单Sheet完全匹配 → 直接合并
     - 多Sheet完全匹配 → 弹窗选择
     - 单Sheet不完全匹配 → 弹窗确认字段映射
     - 多Sheet不完全匹配 → 弹窗选择Sheet+确认字段映射

#### 需求2：优化错误处理
**优先级**: P0 (关键)

**详细需求**:
- 未匹配到的数据字段默认为 0.00
- 不报错中断流程
- 在结果中标记未匹配字段

#### 需求3：优化错误提示
**优先级**: P1 (高)

**详细需求**:
- 将技术错误信息转换为中文可读信息
- 分类错误类型（文件错误、数据错误、系统错误）
- 提供具体的解决建议

#### 需求4：新增输出位置设置
**优先级**: P1 (高)

**详细需求**:
- 生成报表时弹窗选择输出位置
- 设置页面可配置默认输出路径
- 记住用户上次选择的路径
- 路径持久化存储到数据库
